# MPK Mebel Landing

Промт-ориентированная версия лендинга для `mpk-mebel.ru` с файловой админкой и автодеплоем на сервер.

## Что реализовано

- Сайт пересобран по дизайн-системе из `mpk-prompts.md`: тёмная палитра, латунный акцент, `Playfair Display` для заголовков и `Manrope` для интерфейса.
- Структура стала чище и продающее: hero -> категории -> проекты -> галерея -> преимущества -> этапы -> о компании -> акции -> отзывы -> FAQ -> финальный CTA -> контакты.
- Анимации сведены к аккуратным fade-in/reveal через `IntersectionObserver`, без лишнего визуального шума.
- Контент, контакты и изображения можно менять через файловую админку без правки шаблона вручную.

## Важные упрощения относительно промтов

- Блок `Галерея` выполнен как статичная сетка, а не marquee-лента.
- Блок `Этапы` выполнен как чистый timeline без анимации заполнения линии.

Так сделано специально: эти два блока были самыми рискованными с точки зрения адаптива и визуальной стабильности. Упрощение оставляет стиль промтов, но убирает хрупкое поведение.

## Архитектура

- `site.template.html` — основной шаблон сайта, который хранится в репозитории.
- `index.php` — точка входа публичного сайта.
- `storage/site.html` — активная HTML-версия сайта на сервере.
- `admin.html` — интерфейс админки.
- `admin.js` — схема полей админки и логика сохранения.
- `admin.css` — стили админки.
- `api/*.php` — серверные endpoints для логина, сохранения HTML и загрузки изображений.
- `storage/uploads/` — пользовательские изображения, загруженные из админки.

## Структура шаблона

- `header` — бренд, навигация, быстрые контакты.
- `hero` — главный оффер, CTA и нижняя полоса со статистикой.
- `intro-note` — категории мебели в двух рядах.
- `projects` — портфолио в bento-сетке.
- `gallery` — дополнительная сетка реальных работ.
- `advantages` — sticky-блок с преимуществами.
- `process` — 4 этапа работы.
- `story` — блок о компании и мини-метрики.
- `offers` — спецпредложения.
- `reviews` — слайдер отзывов с точками.
- `faq` — аккордеон частых вопросов.
- `final-cta` — центральный оффер перед контактами.
- `contacts` — контактные данные и форма заявки.
- `footer` — навигация, контакты, мессенджеры.

## Как работает админка

- Админка редактирует HTML-документ по CSS-селекторам, описанным в `admin.js`.
- При сохранении обновлённый HTML целиком записывается в `storage/site.html`.
- Изображения можно либо вставлять URL-адресом, либо загружать через `api/upload.php`.
- Если `storage/site.html` отсутствует, публичный сайт автоматически отдаёт `site.template.html`.

## Что важно при доработках

- Если меняете разметку секций, проверяйте соответствующие селекторы в `admin.js`.
- Для новых визуальных блоков держите систему: фон `#0F0E0C`, текст `#F5F0EB`, акцент `#B8956A`, без теней и крупных скруглений.
- Не храните секреты в репозитории: `api/config.php` должен оставаться только на сервере.
- Не перетирайте runtime-папку `storage/` при деплое.

## Деплой

- При `push` в `main` GitHub Actions запускает деплой на сервер.
- Деплой идёт по SSH-ключу через пользователя `deploy`.
- Публичный сайт обслуживается сервером с `index.php`, поэтому после деплоя важно синхронизировать и шаблон, и runtime-версию, если нужна мгновенная публикация текущего состояния.
